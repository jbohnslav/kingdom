# kd council show should default to most recent thread, not longest

## Context

`kd council show` without an ID currently shows the thread with the most
messages, not the most recently created one. When you just fired off a new
question and want to check on it, you see the old thread instead.

## Acceptance Criteria

- [x] `kd council show` (no args) defaults to the most recently created thread
- [x] If there's an active watch/async thread, prefer that one
